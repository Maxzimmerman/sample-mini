What is a React function component, in one sentence?	A plain function that takes a props object and returns JSX. No <code>class</code>, no <code>extends</code>, no <code>render()</code> method — React calls the function and turns the returned JSX into DOM.	stage2::components-mui
Where do you put a component's props interface in mosaic-web style?	Right next to the component, in the same file. Domain types (e.g. <code>Book</code>) live in <code>types.ts</code>; component-specific prop shapes do not.	stage2::components-mui
How do you type and destructure props on a function component?	<code>interface HelloProps { name: string; loud?: boolean }</code><br><code>export function Hello({ name, loud }: HelloProps) { ... }</code>	stage2::components-mui
What does JSX <code>&lt;Typography variant="h6"&gt;Hello&lt;/Typography&gt;</code> compile to?	<code>React.createElement(Typography, { variant: 'h6' }, 'Hello')</code>. Attributes become props, content between tags becomes children.	stage2::components-mui
What do the curly braces in JSX (e.g. <code>{book.title}</code>) do?	They are an escape hatch back to a TS expression. Anything inside <code>{ ... }</code> is evaluated as code; anything outside is treated as literal markup.	stage2::components-mui
JSX attribute syntax: which of <code>color=success</code>, <code>color="success"</code>, <code>color={success}</code> are valid?	<code>color="success"</code> (string literal) and <code>color={success}</code> (expression in braces) are valid. <code>color=success</code> is a parse error.	stage2::components-mui
What is MUI's <code>Box</code>?	A styled <code>&lt;div&gt;</code>. Reach for it when you need a single container you want to style via the <code>sx</code> prop.	stage2::components-mui
What is MUI's <code>Stack</code> and why use it over hand-rolled flexbox?	Children laid out in a row or column with consistent spacing.<br><code>&lt;Stack direction="row" spacing={2}&gt;...&lt;/Stack&gt;</code><br>Easier and theme-aware compared to writing flexbox CSS yourself.	stage2::components-mui
What does Typography's <code>variant</code> prop do?	Picks a typographic role from the theme: <code>h1</code>–<code>h6</code>, <code>body1</code>, <code>body2</code>, <code>overline</code>, <code>caption</code>, etc. Controls font size, weight, and line-height consistently.	stage2::components-mui
Which MUI primitives compose a basic card?	<code>&lt;Card&gt;</code> as the surface, <code>&lt;CardContent&gt;</code> for padded body, <code>&lt;CardActions&gt;</code> for button rows.	stage2::components-mui
What is MUI's <code>Chip</code> used for, and what are its two key props?	A small pill, used for tags or statuses. Key props: <code>label</code> (the text) and <code>color</code> (one of <code>'default' | 'primary' | 'secondary' | 'success' | 'error' | 'info' | 'warning'</code>).	stage2::components-mui
What does <code>sx={{ p: 2 }}</code> on an MUI component do?	Applies <code>padding: theme.spacing(2)</code> (16px by default) via the theme. <code>sx</code> shorthands are theme-aware and type-checked — a typo like <code>borderRdius</code> is a TS error.	stage2::components-mui
What do <code>bgcolor: 'background.paper'</code> and <code>color: 'text.secondary'</code> resolve to?	Theme palette lookups. <code>'background.paper'</code> reads <code>theme.palette.background.paper</code>; <code>'text.secondary'</code> reads <code>theme.palette.text.secondary</code>. You get dark-mode and brand changes for free.	stage2::components-mui
When should you reach for <code>styled(...)</code> instead of <code>sx</code>?	Only when a component has lots of custom CSS. Default to <code>sx</code> for one-off styling — that's what mosaic-web does on almost every screen.	stage2::components-mui
How do you render a list of items in JSX?	Map the array into JSX and give each item a <code>key</code>:<br><code>{books.map((b) =&gt; &lt;BookCard key={b.id} book={b} /&gt;)}</code>	stage2::components-mui
Why does React require a <code>key</code> prop on rendered list items?	React uses keys to match old and new items during reconciliation — so it can reuse DOM nodes on re-render instead of rebuilding the list.	stage2::components-mui
Why is using the array index as a <code>key</code> a bug?	Indices shift when the list is sorted, filtered, or items are inserted/removed. React then misaligns state and DOM, causing wrong items to update or stale UI. Use a stable id.	stage2::components-mui
Why prefer path aliases (e.g. <code>@books/components/BookCard</code>) over relative imports?	They stay readable as paths get deep, and they don't break when you move files around. mosaic-web has 78 aliases configured.	stage2::components-mui
Where are TS path aliases defined?	<code>tsconfig.app.json</code> under the <code>compilerOptions.paths</code> map. Vite also needs to know about them (either via <code>vite-tsconfig-paths</code> or <code>resolve.tsconfigPaths</code>).	stage2::components-mui
mosaic-web export convention: named or default exports?	Named exports. Every component file looks the same: imports, props interface, <code>export function ComponentName(...)</code>. Avoid <code>export default</code>.	stage2::components-mui
What is a React hook, and where can it be called?	Any function whose name starts with <code>use</code> (<code>useState</code>, <code>useEffect</code>, custom <code>useFoo</code>). Hooks may only be called inside a component function or another hook — never at module top level, never conditionally.	stage2::components-mui
Console error: <em>"Invalid hook call"</em> on a blank page. Most likely cause?	A hook (e.g. <code>useEffect</code>) was called outside a component — most often at module top level in a file that React imports. Move it inside the component body.	stage2::components-mui
How do you derive Chip's <code>color</code> prop type without re-typing the union?	<code>import { Chip, type ChipProps } from '@mui/material';</code><br><code>const c: ChipProps['color'] = 'success';</code><br>If MUI changes the union, your code stays in sync.	stage2::components-mui
Pattern: pick a Chip <code>color</code> from a domain status. How?	A small helper with an exhaustive switch:<br><code>function statusColor(s: BookStatus): ChipProps['color'] { switch (s) { case 'read': return 'success'; case 'reading': return 'primary'; case 'want_to_read': return 'default'; } }</code>	stage2::components-mui
What's the difference between the <code>label</code> for a status chip and its <code>color</code>?	Two independent mappings from the same status: <code>label</code> is the human string ("Read"), <code>color</code> is a visual token ('success'). Don't reuse one variable for both.	stage2::components-mui
Why does <code>&lt;BookCard book={} /&gt;</code> fail to compile?	An empty expression <code>{}</code> is not a valid JSX attribute value — you must put something inside the braces, e.g. <code>book={SAMPLE_BOOKS[0]}</code>.	stage2::components-mui
Define: Props	The data passed into a component. Functionally, the single object argument of the component function.	stage2::components-mui
Define: JSX	The HTML-looking syntax used in <code>.tsx</code> files. It compiles to <code>React.createElement</code> calls; it is not HTML and is not a string template.	stage2::components-mui
Define: Render (in React)	When React calls your component function and walks the JSX tree it returns. Happens on first mount and any time state or props change.	stage2::components-mui
Define: Reconciliation	The diff React does between two renders to figure out the minimum set of DOM updates. You don't write reconciliation — you write JSX and React does it.	stage2::components-mui
